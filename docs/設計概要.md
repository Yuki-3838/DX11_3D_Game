# DX11 3D Game 設計概要

このプロジェクトは、敵の行動を読んで、攻撃後のすきを突く一対一アクションの土台です。
最初から100体を作るのではなく、100体を安全に増やせるデータ、計測、デバッグの流れを先に固定します。

## 現在の実行フロー

1. `Main.cpp` が `Application` を作り、3Dサンプルの起動経路へ入る。
2. `Application` がWin32ウィンドウを作成し、`gameinit` を呼び出す。
3. `gameinit` が `Renderer`、DirectInput、ImGui、`SceneManager` を初期化する。
4. `SceneManager` が `GameScene` を生成し、カメラ、フィールド、プレイヤー、敵を初期化する。
5. `gameupdate` が入力とシーンを更新し、`gamedraw` がカメラ、モデル、衝突デバッグ表示を描画する。
6. モデルは `CStaticMesh`／Assimp、GPU描画は `CStaticMeshRenderer` とシェーダーが担当する。
7. 終了時に `DebugUI`、`SceneManager`、`Renderer` の順でリソースを解放する。

シーン切り替えは `F1` で `GameScene`、`F2` で `CarScene` です。切り替え時には現在のシーンを破棄してから次のシーンを初期化します。`GameScene` は `WallScene` の初期内容を引き継いだ、就職作品用の作業シーンです。

`CarScene` では、参照プロジェクトのモデルローダーを移植しています。ImGuiの`Model Selector`から14種類のモデルを選択でき、未ロードのモデルは選択時に`CStaticMesh`とRendererへ登録されます。`Bounding Box`ウィンドウではモデル同士のAABB／OBB判定も確認できます。

## GameScene 1v1 prototype

`GameScene` はプレイヤー1人と敵1体の対戦に整理しています。敵の自動接近、攻撃の予備動作・攻撃判定・硬直、ガードによるダメージ軽減、HPと勝敗状態を `OneVsOneCombat` で管理します。

プレイヤーモデルは `assets/model/Furina/【芙宁娜】.pmx` を読み込みます。GameSceneでは壁を生成せず、フィールド上でプレイヤーと敵が直接対戦する構成です。Furinaフォルダの同梱readmeに商用利用不可の条件があるため、作品公開・提出前に利用許諾を確認してください。

- `WASD`: プレイヤー移動
- `← / →`: プレイヤー回転
- `Space / J`: 攻撃
- `K / マウス右`: ガード
- `R`: 対戦リセット

画面の `1v1 Combat` ウィンドウで、現在の状態と両者のHPを確認できます。

`GameApp` 側の戦闘プロトタイプと計測コードもプロジェクトには残してあり、3Dシーンへ戦闘処理を移植する際に再利用できます。

## 戦闘データの考え方

`CombatDesign.h` は、攻撃を以下の区間で扱います。

- `Anticipation`: 前兆。プレイヤーが読んで回避やガードを選ぶ時間。
- `Active`: 攻撃判定が出る時間。
- `Recovery`: 反撃のすき。
- `Cooldown`: 次行動までの制限時間。
- `Commit`: 一度始めると中断できない時間。

命中判定は2段階です。

- `BroadPhaseFilter`: 距離、角度、状態で候補を絞る。
- `HitboxDefinition` と `NarrowPhaseType`: 球、カプセル、OBBなどで精密判定する。

この構造にしておくと、敵ごとの攻撃やAIの違いをコード分岐ではなくデータ差分で増やせます。

## デバッグ表示

Dear ImGuiは `ImGuiDebugAdapter` に閉じ込めています。
持ち込んだ `system/imgui` の本体とDX11/Win32バックエンドをプロジェクトへ登録し、
`DX11_GAME_ENABLE_IMGUI` を有効化済みです。`DebugOverlay` の値がゲーム画面内に表示されます。

ブラウザ確認はゲーム起動中に生成される `debug/index.html` を開きます。
同じフォルダに更新される `debug/debug_state.json` を読み、FPS、フレーム時間、攻撃フェーズ、命中候補数を表示します。

## Phase 2 Prototype Controls

Phase 2の入口として、DX11だけで戦闘状態を見られる矩形デバッグビューを追加しています。

- `A`: プレイヤーを左へ移動
- `D`: プレイヤーを右へ移動
- `Shift`: ガード状態
- `Space`: 敵のサンプル攻撃を最初から再生
- `Esc`: 終了

画面上では、青がプレイヤー、赤が敵、茶色が前兆範囲、オレンジがActive中の攻撃判定です。
敵がRecovery中のときは敵表示が黄色寄りになり、反撃できる「すき」を示します。

ブラウザデバッグの `debug/index.html` には、FPS、フレーム時間、攻撃フェーズ、距離、ガード状態、命中候補数、命中数が出ます。

Phase 2以降は、描画後に1フレームが16.67msへ近づくよう待機して、常時60FPS固定を前提に計測します。
重い処理が入って16.67msを超えた場合は待機できないため、ブラウザデバッグのFrame msで予算超過を確認します。
